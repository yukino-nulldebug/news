Portfolio Summary

外部データ連携を、公開できるCLIバッチとして仕上げたプロジェクト

RSS/APIから国内外ニュースとマーケット情報を取得し、Markdownレポートへ保存するPythonアプリです。 見せたい力は、機能の多さではなく、再現性、失敗時の扱い、ログ、秘密情報保護、テストまで含めて小さなシステムを完成させる力です。

  • Python CLI
  • RSS/API
  • Markdown output
  • pytest 85 tests
  • GitHub Actions
  • Secret masking
  • AI-assisted dev

AI-Assisted Development

AIを使ってコードを書き、レビューで品質を上げる進め方

AIは設計整理、実装案、コードレビュー、テスト観点の洗い出しに使い、採用する仕様、安全性、最終差分は自分で判断しました。 「AIに作らせた」ではなく、AIを開発工程に組み込み、検証可能なコードへ落とす力を見せる構成です。

Prompt

要件を実装単位へ分解

朝の情報収集という課題から、実行モード、データ構造、例外方針、受け入れ条件へ落とし込む。

Build

AI支援でコードとテストを作成

Provider、設定、レポート生成、pytestを小さな単位で実装し、差分を確認しながら採用する。

Review

AIレビューを品質ゲートにする

外部データ全滅時の判定漏れなど、仕様の穴をレビューで見つけ、修正方針へ変換する。

Verify

指摘を回帰テストで固定

修正だけで終わらせず、異常系テストを追加して pytest とCIで確認できる状態にする。

AIで加速したこと 自分で判断したこと 成果物
要件、設計、実装方針の候補出し APIキーなしで見せるため sample モードを標準にする判断 要件定義、基本設計、settings.py
コードレビューとセキュリティ観点の洗い出し 外部データ全滅を成功扱いにしない仕様判断 main.py、test_api_mode.py
READMEやHTML表紙の構成案 何を評価してほしいかを5軸に整理する判断 public/index.html、docs/portfolio-summary.md

Design Decisions

設計判断と実装の証拠

課題 判断 実装・確認ポイント
APIキーがない閲覧者が試せない sample モードを標準経路にする APP_MODE 未設定時の設定読み込み、サンプルJSON、サンプルレポート
外部取得失敗を成功に見せてしまう 部分失敗は警告、全滅は終了コード 2 build_report_data() と test_api_mode.py
無料APIの制限に当たりやすい 連続取得間隔を .env で調整可能にする MARKET_REQUEST_INTERVAL_SECONDS とProviderテスト
公開リポジトリで秘密情報が漏れる URL、警告、ログ、レポートの値をマスクする sanitize_url()、sanitize_text()、security tests
