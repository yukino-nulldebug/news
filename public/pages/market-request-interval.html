Change

背景

Alpha Vantage の無料枠では短時間の連続リクエストで制限に当たることがある。 複数のマーケット対象を取得するとき、各取得の間で待機できる設定を追加した。

timeout

10s

retry

4

market interval

2s

# external request
REQUEST_TIMEOUT_SECONDS=10
REQUEST_RETRY_COUNT=4
MARKET_REQUEST_INTERVAL_SECONDS=2

Implementation

実装内容

ファイル変更内容
morning-news/.envMARKET_REQUEST_INTERVAL_SECONDS=2 を追加し、誤った単数形の変数名を置き換えた。
morning-news/.env.exampleAlpha Vantage向けの推奨外部取得設定を追加した。
src/config/settings.pyMARKET_REQUEST_INTERVAL_SECONDS を読み込み、Settings.market_request_interval_seconds として保持する。
src/market/providers/alpha_vantage.py複数ターゲット取得ループで、最後以外の取得後に time.sleep() する。
tests/設定読み込みと待機処理のテストを追加した。

Operation

運用メモ

最初の推奨値

MARKET_REQUEST_INTERVAL_SECONDS=2 で試す。

制限に当たる場合

MARKET_REQUEST_INTERVAL_SECONDS=5 へ増やして再実行する。

未設定時のコード既定値は 0 で、待機しない。
