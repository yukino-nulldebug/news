Phase 1 のゴール

Phase 1 は、外部APIを使わずに sample_data の固定JSONから日次Markdownレポートを生成し、 reports/YYYY-MM-DD.md として保存できる状態を作る。

入力

整形・計算

出力

sample_data JSON仕様

ニュースは地域区分を region、内容分類を category に分ける。サンプルデータは架空データとして扱う。

news_jp.json

{
  "items": [
    {
      "region": "domestic",
      "category": "economy",
      "title": "国内ニュースのサンプルタイトル",
      "url": "https://example.com/jp-news-1",
      "source": "Sample JP News",
      "published_at": "2026-05-19T07:00:00+09:00",
      "summary": "国内経済に関するニュース概要のサンプルです。"
    }
  ]
}

news_global.json

{
  "items": [
    {
      "region": "global",
      "category": "market",
      "title": "Global market sample news title",
      "url": "https://example.com/global-news-1",
      "source": "Sample Global News",
      "published_at": "2026-05-19T07:00:00+09:00",
      "summary": "This is a sample summary for global market news."
    }
  ]
}

market.json

{
  "items": [
    {
      "symbol": "NIKKEI225",
      "name": "日本市場参考指標",
      "current_value": 38500.25,
      "previous_close": 38200.00,
      "unit": "points",
      "fetched_at": "2026-05-19T07:00:00+09:00"
    }
  ]
}

08

関数設計

main.py

  • main(): 全体処理を実行し、終了コードを返す。
  • build_report_data(settings): 取得、整形、計算結果を ReportData にまとめる。

config/settings.py

  • load_settings(): Phase 1 の設定値を返す。
  • NEWS_LIMIT: 国内/海外ニュースの最大件数。初期値は 5。
  • SUMMARY_MAX_LENGTH: 概要文の最大文字数。初期値は 120。

news/

  • load_news_items(file_path): JSONを読み込み、ニュース一覧を返す。
  • fetch_sample_news(settings): 国内/海外ニュースを返す。
  • format_summary(summary, max_length): 空白整理と文字数制限を行う。
  • format_news_items(items, max_length): short_summary を追加する。

market/

  • load_market_items(file_path): market.json を読み込む。
  • fetch_sample_markets(settings): Phase 1 用の市況データを返す。
  • calculate_change(current_value, previous_close): 前日比と変化率を計算する。
  • calculate_market_changes(items): 各項目に計算結果を追加する。

report/

  • generate_report(report_data): Markdown全文を生成する。
  • generate_news_section(title, items): ニュースセクションを生成する。
  • generate_market_section(items): 市況表を生成する。
  • generate_market_comments(items): 市況コメントを生成する。

writer / logger

  • build_report_path(report_dir, target_date): 保存先を作る。
  • write_report(markdown, report_path): UTF-8で保存する。
  • setup_logger(log_dir): 標準出力と logs/app.log に出力する。

09

計算・整形ルール

概要文整形

  • \n, \r, \t は半角スペースに置換する。
  • 連続する空白は1つにまとめる。
  • 前後の空白は削除する。
  • max_length を超える場合は末尾を ... にする。
  • 空の場合は 概要はありません。 とする。

変化率計算

change = current_value - previous_close
change_rate = change / previous_close * 100
  • change と change_rate は小数第2位まで丸める。
  • previous_close が 0 の場合は None とする。

    squash --> limit["120文字以内"]
    limit --> short["short_summary"]

    current["current_value"] --> change["change"]
    previous["previous_close"] --> change
    change --> rate["change_rate"]

エラー処理

flowchart TD
    start["処理中にエラー"] --> canContinue{"継続できるか"}
    canContinue -->|Yes| warning["WARNINGログ"]
    warning --> fallback["代替値 / N/A / 空概要"]
    fallback --> continue["処理継続"]
    canContinue -->|No| error["ERRORログ"]
    error --> stop["終了コード1"]

    missingJson["JSONなし"] --> error
    invalidJson["JSON形式不正"] --> error
    missingRequired["必須項目欠損"] --> error
    emptySummary["summary空"] --> warning
    zeroPrevious["previous_close = 0"] --> warning
処理停止
  • JSONファイルが存在しない
  • JSON形式が不正
  • items が存在しない
  • 必須項目が欠損している
処理継続
  • summary が空
  • previous_close が 0
  • logs/app.log に書き込めない
終了コード
0正常終了
1生成または保存失敗

12

テスト観点と受け入れ条件

単体テスト

  • format_summary(): 空白整理、文字数制限、空概要
  • calculate_change(): 正常値、previous_close=0
  • generate_market_section(): Markdown表生成
  • build_report_path(): 日付付き保存先

結合・異常系テスト

  • python main.py が終了コード 0 で終わる。
  • reports/YYYY-MM-DD.md が作成される。
  • JSON欠損、不正JSON、必須項目欠損で終了コード 1 になる。
  • 再実行時に同一日付ファイルを上書きできる。
APIキー不要sample のみで実行

保存reports/YYYY-MM-DD.md
構成6つの主要セクション
