05

4.4 文字数上限と省略ルール

max_length は省略記号を含めた最終文字数の上限とする。Phase 3 の仮置き値は 120。

文字数の考え方

11020304050 60708090117...
  • max_length >= 3: 本文を max_length - 3 文字にして ... を付ける。
  • max_length < 3: . を max_length 個返す。
  • max_length <= 0: SummaryFormatError とする。

処理分岐

flowchart TD
    A["format_summary(summary, max_length)"] --> B{"max_length is valid?"}
    B -->|No| E["SummaryFormatError"]
    B -->|Yes| C["空白を正規化"]
    C --> D{"空文字?"}
    D -->|Yes| F["概要はありません。"]
    D -->|No| G{"上限以内?"}
    G -->|Yes| H["そのまま返す"]
    G -->|No| I{"max_length >= 3?"}
    I -->|Yes| J["本文 + ..."]
    I -->|No| K[". を返す"]

4.7 概要文欠損時の警告

欠損検出は main.py に統一する。formatter.py は代替文を返すだけで、ログや集計を更新しない。

責務分担

sequenceDiagram
    participant M as main.py
    participant F as formatter.py
    participant R as execution_result.py
    participant L as logger.py

    M->>M: 整形前の summary を確認
    alt summary が空
      M->>R: record_warning(F-03)
      M->>L: WARNING F-03
    end
    M->>F: format_news_items(items, max_length)
    F-->>M: short_summary 付き list

実装方針

for index, item in enumerate(domestic_news):
    if not str(item.get("summary", "")).strip():
        record_recoverable_warning(
            logger,
            result,
            "F-03",
            "news.formatter",
            f"domestic_news[{index}] の summary が空のため代替文を設定しました",
        )

5. マーケット変化率計算

current_value と previous_close から前日比と変化率を計算し、各マーケット項目へ追加する。

前日比

change = current_value - previous_close

日経平均: 38500.25 - 38200.00 = 300.25

変化率

change_rate = change / previous_close * 100

300.25 / 38200.00 * 100 = 0.79

丸め

round(value, 2)

表示は符号付き小数第2位: +0.79%

8. エラー処理とログ

継続可能な警告

  • summary が空: 概要はありません。 を設定して継続。
  • previous_close == 0: N/A を表示して継続。
  • レポート保存まで完了した場合、終了コードは 0。
  • 実行結果ステータスは warning。

停止するエラー

  • SUMMARY_MAX_LENGTH <= 0: SummaryFormatError。
  • マーケット計算に必要な項目が欠損。
  • 価格データが数値ではない。
  • レポート生成時に必須キーが欠損。
stateDiagram-v2
    [*] --> Running
    Running --> Warning: summary empty / previous_close zero
    Warning --> Saved: 代替値で継続
    Running --> Failed: SummaryFormatError / MarketCalculationError
    Saved --> Exit0: status warning or success
    Failed --> Exit1: status failed
    Exit0 --> [*]
    Exit1 --> [*]

11. 関数設計

main.py

  • build_report_data(): ニュース整形、変化率計算、警告記録、件数集計をまとめる。
  • record_recoverable_warning(): 継続可能な警告をログと実行結果に記録する。
  • 整形前のニュースから概要文欠損を検出する。

news/formatter.py

  • format_summary(summary, max_length): 空白正規化、空概要代替、文字数上限を適用する。
  • format_news_items(items, max_length): 各ニュースへ short_summary を追加する。
  • 警告情報は返さない。

market/calculator.py

  • _is_number(value): bool を除外して数値判定する。
  • calculate_change(): 前日比と変化率を返す。
  • calculate_market_changes(): 各項目へ計算結果を追加する。

report/generator.py

  • generate_news_section(): short_summary を表示する。
  • generate_market_section(): change と change_rate を表示する。
  • generate_market_comments(): 各マーケットにつき1件のコメントを生成する。
