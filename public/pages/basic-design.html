01

設計サマリー

Morning News は、国内外ニュースとマーケット情報を取得し、日次Markdownレポートとして保存するCLIバッチアプリケーションである。

基本設計では、実行モード、データ正規化、レポート生成、ログ・エラー処理、実装フェーズを整理し、要件から実装へ移れる状態を作る。

実行方式

CLIバッチ

標準モード

sample

出力

Markdown

保存先

reports/

02

システム全体構成

03

実行モード設計

sample APIキーなしで確認

sample_data/*.json からニュースと市況データを読み込む。ポートフォリオ公開時の標準動作。

APP_MODE
未設定時は sample
api 実データ取得

RSS/APIからデータを取得する。APIキーは .env から読み込み、ログやレポートには出さない。

モード判定ルール
APP_MODE 動作 ログ
未設定 sample モードとして実行する。 INFO
sample 固定JSONを読み込んで実行する。 INFO
api API/RSSからデータ取得して実行する。 INFO
その他 sample モードへフォールバックする。 WARNING

04

処理フロー

flowchart TD
    start([開始]) --> config["設定読み込み"]
    config --> logging["ログ初期化"]
    logging --> mode{"sample mode?"}
    mode -->|Yes| sample["固定JSONを読み込む"]
    mode -->|No| api["API / RSSからデータ取得"]
    sample --> news["ニュース取得完了"]
    api --> news
    news --> market["マーケット情報取得"]
    market --> format["概要文整形"]
    format --> calc["前日比・変化率計算"]
    calc --> comment["市況コメント生成"]
    comment --> markdown["Markdown生成"]
    markdown --> save["reports/YYYY-MM-DD.md に保存"]
    save --> log["実行結果をログ出力"]
    log --> finish([終了])
Step 1 設定読み込み

.env と既定値から実行設定を作る。

Step 2 モード判定

sample / api の取得元を決める。

Step 3 データ取得

国内ニュース、海外ニュース、市況データを取得する。

Step 4 整形・計算

概要文整形、前日比、変化率を処理する。

Step 5 Markdown生成

注意事項と市況コメントを含めて組み立てる。

Step 6 保存・ログ

レポート保存先と成功/失敗件数を記録する。

異常時の基本方針

  • ニュース取得失敗時は、該当カテゴリだけ欠損扱いにして処理を継続する。
  • マーケット情報の一部欠損は N/A として表示し、理由をログに残す。
  • レポート生成または保存に失敗した場合は、異常終了として扱う。

05

データ取得・正規化

レポート生成処理が取得元の違いを意識しないように、取得後のデータを共通形式に正規化する。

ニュース正規化

  • category: domestic / global
  • title, url, source は必須
  • published_at はJST表示に変換
  • short_summary は120文字以内に整形

マーケット正規化

  • current_value と previous_close から前日比を計算
  • change_rate は change / previous_close * 100
  • 欠損値は N/A として出力可能にする
  • 対象は日経平均、S&P500、USD/JPYを最低範囲とする

07

sample_data JSON仕様

サンプルデータは架空データとし、実在ニュースの本文や要約文をコピーしない。APIキーなしで安全に動作確認できるようにする。

      "title": "国内ニュースのサンプルタイトル",
      "url": "https://example.com/jp-news-1",
      "source": "Sample JP News",
      "published_at": "2026-05-16T07:00:00+09:00",
      "summary": "国内経済に関するニュース概要のサンプルです。"
    }
  ]
}

      "published_at": "2026-05-16T07:00:00+09:00",

08

レポート生成設計

出力ルール

  • 保存先は reports/YYYY-MM-DD.md
  • 文字コードはUTF-8
  • MVPでは同一日付ファイルを上書き
  • 欠損データは N/A または取得失敗メッセージで表示
  • 投資助言ではない旨の注意事項を必ず表示
# Morning News Report
作成日時: YYYY-MM-DD HH:mm JST
実行モード: sample/api

## 1. 今日の注目ポイント
## 2. 国内ニュース
## 3. 海外ニュース
## 4. マーケット情報
## 5. 市況コメント
## 6. 注意事項
本レポートは情報提供を目的としており、投資助言ではありません。

09

ログ・エラー処理

ログ設計

  • 出力先は logs/app.log と標準出力
  • 形式は 日時 レベル 機能ID 処理名 メッセージ
  • INFO: 起動、モード判定、取得件数、保存先
  • WARNING: 一部取得失敗、欠損、フォールバック
  • ERROR: 全取得失敗、生成失敗、保存失敗

終了コード

コード 意味
0正常終了
1レポート生成または保存失敗
2データ取得が全滅

11

設定・環境変数

APP_MODE=sample
NEWS_API_KEY=
MARKET_API_KEY=
REPORT_DIR=reports
LOG_DIR=logs
NEWS_LIMIT=5
SUMMARY_MAX_LENGTH=120
REQUEST_TIMEOUT_SECONDS=10
REQUEST_RETRY_COUNT=1

設定方針

  • APP_MODE 未設定時は sample
  • APIキーは空でも公開リポジトリとして成立する
  • 出力先、件数、文字数、タイムアウト、リトライ回数は環境変数で調整可能にする
  • APIキーや認証ヘッダーはログ・レポートへ出力しない

12

実装フェーズ

Phase 1 sample_dataからMarkdown生成

APIキーなしで reports/YYYY-MM-DD.md を作成する。

Phase 2 ログ・エラー処理

実行結果と失敗理由を logs/app.log に出力する。

Phase 3 整形・計算

概要文整形と市況の変化率計算を追加する。

Phase 4 RSS/API取得

APP_MODE=api で外部データを取得する。

Phase 5 pytest追加

主要処理のテストを追加し、回帰を防ぐ。

Phase 6 README・サンプル整備

第三者が手順通りに実行できる状態にする。

13

要件対応とテスト方針

主要要件と確認観点
要件ID 設計上の対応 テスト観点
F-01/F-02 国内・海外ニュースを NewsItem に正規化する。 各カテゴリで1件以上取得できる。
F-03 概要文を120文字以内の short_summary に整形する。 改行・重複空白が除去される。
F-04/F-05 市況データを MarketItem に正規化し、変化率を計算する。 現在値、前日比、変化率が出力される。
F-06〜F-08 Markdownを生成し、日付付きファイルとして保存する。 必須セクションと注意事項が含まれる。
F-09/F-10 失敗理由をログに残し、sample モードで実行可能にする。 APIキーなしでレポート生成できる。
