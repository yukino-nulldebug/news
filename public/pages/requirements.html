01

目的と概要

Morning News は、朝の情報収集にかかる時間を減らすため、国内外ニュースとマーケット情報を自動収集し、 日次Markdownレポートとして保存するバッチ型アプリケーションである。

MVPでは本文全文の解析やAIによる高度な要約は行わず、API/RSSから取得できる概要文を整形して掲載する。

利用シーン

毎朝のニュース・市況確認

成果物

reports/YYYY-MM-DD.md

MVPの中心価値

1つのMarkdownで短時間に確認

02

ポートフォリオで示すこと

設計力

  • 要件定義、機能分割、受け入れ条件を一貫して設計できる
  • ニュース取得、市況取得、Markdown出力の責務を分けて考えられる
  • 投資助言と誤認されない情報設計ができる

実装・公開力

  • API/RSS取得、データ整形、ログ記録を小さなアプリにまとめる
  • APIキーなしでも sample モードで動作確認できる
  • .env やAPIキーを公開しないGitHub構成にする

03

想定ユーザーと課題

想定ユーザー

  • 毎朝、国内外ニュースと市場動向を短時間で把握したい個人ユーザー
  • 業務開始前に「前日から当朝までの要点」を確認したいビジネスユーザー
  • API連携・バッチ処理・Markdown出力の実装例を確認したい閲覧者

課題と解決方針

  • 複数サイト巡回が長い: 1つのMarkdownに集約する
  • ニュースと株価を同時に見づらい: ニュースの後に市況を配置する
  • 過去分を参照しづらい: 日付ベースで保存する
  • APIキーなしで確認しづらい: 固定JSONの sample モードを用意する

04

MVPスコープ

MVPでやる

  • ニュースAPI/RSSから記事情報を取得する
  • 概要文を整形し、短い概要として加工する
  • 株価・指数・為替の現在値、前日比、変化率を扱う
  • 市況コメントを定型ルールで追加する
  • 日次Markdownレポートを生成・保存する
  • sample モードとエラーログを用意する

MVPではやらない

  • AIによる高度な要約
  • ニュース本文全文の解析
  • 有料記事やログイン後ページの本文取得
  • ユーザーごとの高度な推薦アルゴリズム
  • 自動売買シグナル、売買推奨、投資助言
  • Web UI、モバイル通知、スマホアプリ化

成功条件

MVPの合格ライン
観点 成功条件
レポート生成 毎朝1回、手動または定時実行でレポートを生成できる。
情報の網羅性 国内ニュース、海外ニュース、マーケット情報が1つのMarkdownにまとまる。
安全性 市況コメントに投資助言ではない注記が表示され、機密情報がGitHubに含まれない。

第三者確認 README手順とサンプルレポートから、目的・出力・実行方法を理解できる。

07

機能要件

MVP機能一覧
ID 優先度 機能 要件
F-01/F-02 MUST ニュース取得 国内・海外カテゴリの記事について、タイトル・URL・配信元・公開日時・概要文を保持する。
F-03 MUST 概要文整形 不要改行・重複空白を除去し、文字数上限内の短い概要を生成する。
F-04/F-05 MUST 市況取得・変化率計算 日経平均、S&P500、USD/JPY の現在値・前日比・変化率を扱う。
F-06/F-08 MUST レポート生成・保存 ニュース情報、主要市況データ、市況コメント、注意事項を含むMarkdownを保存する。
F-07 SHOULD 市況コメント 情報提供に限定し、投資助言と誤認される表現を使わない。
F-09/F-10 SHOULD/MUST ログ・サンプル実行 失敗理由をログに残し、APIキーなしでもサンプル実行できる。

08

データ取得方針

取得対象と扱わない内容
データ種別 取得方法 MVPで扱う内容 扱わない内容
国内ニュース RSS / News API タイトル、URL、配信元、公開日時、概要文 本文全文、有料記事、ログイン後ページ
海外ニュース RSS / News API / GDELT等 タイトル、URL、配信元、公開日時、概要文 本文全文、自動翻訳全文保存
マーケット情報 株価API 現在値、前日終値、前日比、変化率 自動売買、売買推奨
サンプルデータ JSONファイル APIなしでも動作確認できる固定データ 実運用データとしての利用

09

実行モード

モード

モード 用途 APIキー
sample ポートフォリオ確認用 不要
api 実データ取得用 必要

要件

  • APP_MODE=sample では固定JSONからレポート生成できる
  • APP_MODE=api では .env のAPIキーを使う
  • 未設定時は sample モードで安全に起動する

10

非機能要件・制約

運用品質

  • 朝の定時バッチ1回あたり5分以内を目標とする
  • 成功/失敗件数、失敗理由、最終実行時刻をログで追跡できる
  • データソース、カテゴリ、対象指標を設定ファイルで変更可能にする

安全性・再現性

  • APIキー等の機密情報は環境変数で管理し、レポートへ出力しない
  • 市況コメントは参考情報に限定し、投資助言と誤認される表現を避ける
  • APIキーがなくても sample モードで主要機能を確認できる

11

受け入れ条件

機能別受け入れ条件
対象 条件
F-01/F-02 各カテゴリで最低1件以上、タイトル・URL・配信元・公開日時・概要文を取得できる。
F-03 概要文が文字数上限内に整形され、不要改行や重複空白が除去される。
F-04/F-05 3指標すべてで現在値と前日比（%）が算出され、欠損時は理由がログに残る。
F-06/F-07 ニュース情報、マーケット概要、市況コメント、投資助言ではない旨の注意事項が含まれる。
F-08/F-09/F-10 日付付き保存、失敗ログ、APIキーなしのサンプル生成を確認できる。

技術スタック

採用技術と理由
区分 技術 採用理由
言語 Python 3.x API取得、ファイル出力、バッチ処理に向いているため。
取得 requests / feedparser APIとRSSをシンプルに扱えるため。
