02

2. Phase 6 の対象範囲

Phase 6 で実装する

  • READMEに概要、機能、セットアップ、sample実行、api実行（任意）を記載する。
  • .env.example に公開可能な環境変数を整理する。
  • reports/sample-report.md を固定の出力例として整備する。
  • .gitignore で機密情報、ログ、生成物を除外する。
  • 公開前チェックとREADMEコマンド検証を定義する。
  • 設計ドキュメント、テスト、サンプル出力への導線を用意する。

Phase 6 では実装しない

  • 新しいニュースProviderの追加
  • 実APIキー付きのデモ公開
  • 定時実行の本体実装
  • GitHub Actions必須化
  • Web UI、通知、AI要約
  • 投資助言・売買判断

標準確認

ポートフォリオ確認では sample モードを推奨する。

任意確認

api モードはAPIキーや外部サービス状態で結果が変わる。

安全性

.env、APIキー、個人URL、有料記事本文を公開しない。

再現性

第三者がAPIキーなしでセットアップから出力確認まで進める。

4. セットアップと実行手順

1Clone / cd

cd morning-news

2venv

python3 -m venv .venv

3Install

python3 -m pip install -r requirements.txt

4sample

APP_MODE=sample python3 main.py

5pytest

python3 -m pytest

sampleモード

APIキーなしで最初に確認する標準手順。reports/YYYY-MM-DD.md と logs/app.log が作成される。

apiモード（任意）

APIキーやRSS、外部サービス状態により取得結果が変わる。ポートフォリオ確認では sample モードを推奨する。

05

5. .env.example 設計

公開リポジトリに含める設定例。実APIキー、個人URL、内部URL、個人情報は入れない。

Execution

APP_MODE=sample を既定として、APIキーなし確認を優先する。

News

NEWS_PROVIDER、RSS URL、NEWS_LIMIT、SUMMARY_MAX_LENGTH。

Market

MARKET_PROVIDER、APIキー欄、Alpha Vantage endpoint、FX設定。

Output / Request

REPORT_DIR、LOG_DIR、timeout、retry回数、マーケットAPI取得間隔。

# execution mode
APP_MODE=sample

# news
NEWS_PROVIDER=rss
NEWS_RSS_URLS=
NEWS_JP_RSS_URLS=
NEWS_GLOBAL_RSS_URLS=
NEWS_API_KEY=
NEWS_LIMIT=5
SUMMARY_MAX_LENGTH=120

# market
MARKET_PROVIDER=alpha_vantage
MARKET_API_KEY=
MARKET_API_ENDPOINT=https://www.alphavantage.co/query

# output
REPORT_DIR=reports
LOG_DIR=logs

# external request
REQUEST_TIMEOUT_SECONDS=10
REQUEST_RETRY_COUNT=4
MARKET_REQUEST_INTERVAL_SECONDS=2

06

6. サンプルレポート設計

追加するファイル

reports/sample-report.md を公開用の固定サンプルレポートとして管理する。

  • 日次生成される reports/YYYY-MM-DD.md とは別にする。
  • APP_MODE=sample の出力をもとに作成する。
  • APIキーや外部RSSの内容に依存しない。
Report

含める内容

  • # Morning News Report
  • 作成日時と実行モード sample
  • 国内ニュース、海外ニュース、マーケット情報
  • 市況コメント、注意事項、投資助言ではない旨
含めないもの理由
APIキー、.env の内容機密情報を含む可能性があるため
実在有料記事本文著作権・利用規約リスクがあるため
ログ全文実行環境情報を含む可能性があるため
売買推奨表現投資助言と誤認されるため

7. .gitignore と公開対象

含める

ソースコード、README、.env.example、.gitignore、sample_data/、tests/、docs/。

出力例

reports/sample-report.md は管理し、日次生成レポートは原則除外する。

除外する

.env、.venv/、__pycache__/、.DS_Store、ログ。

注意する

既存追跡済みファイルは .gitignore 追加だけでは追跡解除されない。

# environment
.env
.env.*
!.env.example

# virtualenv
.venv/
venv/

# python
__pycache__/
*.py[cod]
.pytest_cache/

# logs
logs/*.log
!logs/.gitkeep

# generated reports
reports/*.md
!reports/sample-report.md
!reports/.gitkeep

sampleモード実行、レポート生成、ログ生成、pytest成功。

任意確認

実APIキーを使う確認は公開前チェックの必須条件にしない。
