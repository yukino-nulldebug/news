2. 対象範囲

Phase 4 で実装する

  • Settings dataclass と属性アクセス
  • .env / 環境変数の読み込み
  • APP_MODE=sample/api の分岐
  • RSSニュース取得と NewsItem 正規化

3. モード分岐の考え方

sample はAPIキーなしの確認用として維持し、api は外部取得の成否を明確に扱う。

APP_MODE=sample

  • .env がなくても起動する。
  • sample_data/news_jp.json を読む。
  • sample_data/news_global.json を読む。
  • sample_data/market.json を読む。
APP_MODE

APP_MODE=api

  • RSSからニュースを取得する。
  • マーケットAPIから取得できる指標を読む。
  • 失敗時にsampleへ自動フォールバックしない。
  • 全データ取得失敗なら終了コード 2。

7. HTTP共通処理

Providerは requests.get() を直接呼ばない。通信ルールとマスクを共通化する。

Provider

RSS / Market ProviderがHTTP取得を依頼する。

get_text / get_json

GETのみ。User-Agentを付ける。

Retry

timeout、429、5xx は指定回数まで再試行。

Parse

TextまたはJSONを返す。

Mask

ログ用URLは apikey=*** にする。

ERROR F-04 market.alpha_vantage 外部取得に失敗しました: https://www.alphavantage.co/query?function=GLOBAL_QUOTE&symbol=SPY&apikey=***

8. RSSニュース取得

Phase 4 のニュース取得の必須実装はRSS Provider。News APIは未対応Providerスタブに留める。

1. URL確認

RSS URLが空ならニュース0件と警告を返す。

2. 取得

get_text() でRSS/Atom本文を取得する。

3. 解析

feedparser.parse() を使い、bozo 警告でもentriesがあれば継続する。

4. 正規化

title、url、source、published_at、summary を作る。

5. 整理

URL重複を除外し、公開日時の新しい順に並べる。

6. 制限

NEWS_LIMIT 件に切り詰める。

flowchart LR
    rss["RSS URL"] --> text["get_text"]
    text --> parse["feedparser.parse"]
    parse --> entries["entries"]
    entries --> normalize["NewsItem"]
    normalize --> dedupe["URL重複排除"]
    dedupe --> sort["published_at desc"]
    sort --> limit["NEWS_LIMIT"]
    limit --> formatter["format_news_items"]

10. 部分失敗と終了コード

RSS 1件失敗

他RSSを継続。WARNING F-01/F-02。

ニュース全滅

マーケットがあればレポート継続。

マーケット全滅

ニュースがあればレポート継続。

全データ全滅

保存せず ERROR F-09、終了コード 2。

終了コード条件レポート保存
0正常終了、または警告ありで保存完了あり
1設定不備、整形・計算・生成・保存の継続不可エラー原則なし
2sample 必須データ失敗、または api 外部データ全取得失敗なし
