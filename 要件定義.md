# Morning News 要件定義（MVP）

## 1. 目的

ユーザーが朝の情報収集にかける時間を減らすため、ニュースとマーケット情報を自動収集し、重要情報を要約・分類して毎朝レポートとして保存する。

## 2. 想定ユーザー

- 毎朝、国内外ニュースと市場動向を短時間で把握したい個人ユーザー
- 業務開始前に「前日から当朝までの要点」を確認したいビジネスユーザー

## 3. 解決したい課題

- ニュースと株価の関係を同じレポートで確認できない
- 複数サイト巡回による情報収集時間が長い
- 過去レポートを日付ベースで参照しづらい

## 4. スコープ

### 4.1 MVP対象

- ニュースAPI/RSSから記事情報（タイトル・URL・配信元・概要文）を取得する
- 取得した概要文を整形し、短い概要としてレポート向けに加工する
- タイトル・URL・配信元・短い概要をレポートに掲載する
- 株価・指数・為替の取得と前日比計算を行う
- 投資判断コメント（参考）を定型文で追加する（投資助言は行わない）
- 日次Markdownレポート生成と保存
- 処理失敗時のログ記録

### 4.2 MVP非対象

- AIによる高度な要約
- ニュース本文全文の解析
- ユーザーごとの高度な推薦アルゴリズム
- 自動売買シグナルの生成
- Web UIでの編集機能
- モバイル通知連携
- スマホアプリ化

### 4.3 スコープ境界図

MVP対象範囲:
- ニュースAPI/RSSから記事情報取得
- 概要文の整形（短文化）
- タイトル/URL/配信元/概要の掲載
- 市況データ取得と前日比計算
- 投資判断コメント（参考）の付与
- Markdownレポート生成と保存
- 処理失敗時のログ記録

対象外・将来拡張:
- AIによる高度な要約
- ニュース本文全文の解析
- ユーザーごとの高度な推薦アルゴリズム
- 自動売買シグナルの生成
- Web UIでの編集機能
- モバイル通知連携
- スマホアプリ化

## 5. 機能要件

| ID | 優先度 | 機能 | 要件 |
| --- | --- | --- | --- |
| F-01 | MUST | 国内ニュース取得 | ニュースAPI/RSSから国内カテゴリ記事を取得し、タイトル・URL・配信元・概要文を保持する。 |
| F-02 | MUST | 海外ニュース取得 | ニュースAPI/RSSから海外カテゴリ記事を取得し、タイトル・URL・配信元・概要文を保持する。 |
| F-03 | MUST | 概要文整形 | 取得した概要文を整形し、最大文字数をそろえた短い概要を生成する。 |
| F-04 | MUST | マーケット情報取得 | 日経平均、S&P500、USD/JPY を最低対象として価格を取得する。 |
| F-05 | MUST | 変化率計算 | 前営業日終値を基準として前日比（%）を計算する。 |
| F-06 | MUST | レポート生成 | タイトル・URL・配信元・短い概要、主要市況データを含むMarkdownを生成する。 |
| F-07 | SHOULD | 投資判断コメント（参考） | 定型ルールに基づく参考コメントを追記し、「投資助言ではない」注記を明示する。 |
| F-08 | MUST | レポート保存 | `reports/YYYY-MM-DD.md` 形式で保存する。 |
| F-09 | SHOULD | エラーログ | 取得/変換/保存の各失敗を日時・機能ID付きで記録する。 |

## 6. 受け入れ条件

- F-01/F-02: 各カテゴリで最低1件以上の記事について、タイトル・URL・配信元・概要文を取得できる。
- F-03: 概要文が定義した文字数上限内に整形され、不要改行や重複空白が除去される。
- F-04/F-05: 3指標すべてで現在値と前日比（%）が算出され、欠損時は理由がログに残る。
- F-06: レポートに「タイトル」「URL」「配信元」「短い概要」「マーケット概要」が含まれる。
- F-07: 参考コメントに「本コメントは情報提供であり投資助言ではありません」が必ず表示される。
- F-08: 当日分ファイルが重複せず保存され、再実行時は上書きまたは版管理方針に従う。
- F-09: 失敗時に処理名・発生時刻・原因メッセージを出力する。

## 7. 非機能要件

- 実行時間: 朝の定時バッチ1回あたり5分以内を目標とする。
- 可観測性: 成功/失敗件数と最終実行時刻をログで追跡できること。
- 保守性: データソースやカテゴリを設定ファイルで変更可能にする。
- セキュリティ: APIキー等の機密情報は環境変数で管理し、レポートへ出力しない。
- コンプライアンス: 投資判断コメントは参考情報に限定し、投資助言と誤認される表現を避ける。

## 8. 未決事項

- ニュース/価格データの正式ソースと利用規約の確定
- 概要文の文字数上限（例: 80文字 or 120文字）の最終決定
- 投資判断コメントの定型ルール（閾値・表現テンプレート）の最終決定
- バッチ実行時刻（JST）と再試行回数の最終決定

## 9. 成功条件

本MVPは、以下を満たした場合に成功とする。

- 毎朝1回、手動または定時実行でレポートを生成できる
- 国内ニュース、海外ニュース、マーケット情報が1つのMarkdownにまとまる
- 各ニュースにタイトル・URL・配信元・短い概要が含まれる
- 日経平均、S&P500、USD/JPY の現在値と前日比を確認できる
- 参考コメントに「投資助言ではない」注記が表示される
- 取得失敗時に処理が完全停止せず、失敗理由がログに残る
- `.env` に設定したAPIキーをGitHubへ公開しない
- 生成されたレポートを過去日付ごとに参照できる
