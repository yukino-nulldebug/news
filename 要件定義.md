# Morning News 要件定義（MVP）

## 1. 概要

Morning News は、朝の情報収集にかける時間を減らすため、ニュースとマーケット情報を自動収集し、日次Markdownレポートとして保存するバッチ型アプリケーションである。

本MVPでは、ニュース本文全文やAIによる高度な要約ではなく、ニュースAPI/RSSから取得できる概要文を整形し、タイトル・URL・配信元・公開日時・短い概要をレポートに載せることを優先する。

## 2. ポートフォリオで示すこと

このプロジェクトは、単なるメモ生成ではなく、以下を第三者が確認できるポートフォリオとして作る。

- 要件定義から機能分割、受け入れ条件まで一貫して設計できること
- API/RSS取得、データ整形、Markdown出力、ログ記録を小さなアプリとしてまとめられること
- APIキーがない閲覧者でも `sample` モードで動作確認できること
- `.env` やAPIキーを公開しない前提で、GitHub公開に耐える構成にできること
- 市況コメントは情報提供に限定し、投資助言と誤認されないように扱えること

## 3. 想定ユーザーと課題

### 3.1 想定ユーザー

- 毎朝、国内外ニュースと市場動向を短時間で把握したい個人ユーザー
- 業務開始前に「前日から当朝までの要点」を確認したいビジネスユーザー
- ポートフォリオとして、API連携・バッチ処理・Markdown出力の実装例を確認したい閲覧者

### 3.2 解決したい課題

- ニュースと株価の関係を同じレポートで確認できない
- 複数サイト巡回による情報収集時間が長い
- 過去レポートを日付ベースで参照しづらい
- APIキーがない環境でも成果物の動作を確認しづらい

## 4. 成功条件

本MVPは、以下を満たした場合に成功とする。

- 毎朝1回、手動または定時実行でレポートを生成できる
- 国内ニュース、海外ニュース、マーケット情報が1つのMarkdownにまとまる
- 各ニュースにタイトル・URL・配信元・公開日時・短い概要が含まれる
- 日経平均、S&P500、USD/JPY の現在値と前日比を確認できる
- 市況コメントに「投資助言ではない」注記が表示される
- 取得失敗時に処理が完全停止せず、失敗理由がログに残る
- `.env` に設定したAPIキーをGitHubへ公開しない
- 生成されたレポートを過去日付ごとに参照できる
- APIキーなしでも `sample` モードでレポート生成を確認できる
- READMEの手順どおりに実行すれば、第三者がローカルで動作確認できる
- `.env.example` に必要な環境変数が明記されている
- GitHubに機密情報が含まれていない
- サンプルレポートから、アプリの目的と出力内容が理解できる

## 5. MVPスコープ

### 5.1 MVPでやる

- ニュースAPI/RSSから記事情報（タイトル・URL・配信元・公開日時・概要文）を取得する
- 取得した概要文を整形し、短い概要としてレポート向けに加工する
- タイトル・URL・配信元・公開日時・短い概要をレポートに掲載する
- 株価・指数・為替の取得と前日比計算を行う
- 市況コメント（参考）を定型ルールで追加する
- 日次Markdownレポートを生成して保存する
- APIキーなしで確認できる `sample` モードを用意する
- 処理失敗時のログを記録する

### 5.2 MVPではやらない

- AIによる高度な要約
- ニュース本文全文の解析
- 有料記事やログイン後ページの本文取得
- ユーザーごとの高度な推薦アルゴリズム
- 自動売買シグナルの生成
- 売買推奨や投資助言
- Web UIでの編集機能
- モバイル通知連携
- スマホアプリ化

## 6. 機能要件

| ID | 優先度 | 機能 | 要件 |
| --- | --- | --- | --- |
| F-01 | MUST | 国内ニュース取得 | ニュースAPI/RSSから国内カテゴリ記事を取得し、タイトル・URL・配信元・公開日時・概要文を保持する。 |
| F-02 | MUST | 海外ニュース取得 | ニュースAPI/RSSから海外カテゴリ記事を取得し、タイトル・URL・配信元・公開日時・概要文を保持する。 |
| F-03 | MUST | 概要文整形 | 取得した概要文から不要改行・重複空白を除去し、最大文字数をそろえた短い概要を生成する。 |
| F-04 | MUST | マーケット情報取得 | 日経平均、S&P500、USD/JPY を最低対象として価格情報を取得する。 |
| F-05 | MUST | 変化率計算 | 前営業日終値を基準として前日比と変化率（%）を計算する。 |
| F-06 | MUST | レポート生成 | タイトル・URL・配信元・公開日時・短い概要、主要市況データを含むMarkdownを生成する。 |
| F-07 | SHOULD | 市況コメント（参考） | 定型ルールに基づき、価格変動に対する参考コメントを追記する。ただし、売買推奨・投資助言と誤認される表現は使用しない。 |
| F-08 | MUST | レポート保存 | `reports/YYYY-MM-DD.md` 形式で保存する。 |
| F-09 | SHOULD | エラーログ | 取得/変換/保存の各失敗を日時・機能ID付きで記録する。 |
| F-10 | MUST | サンプル実行 | `APP_MODE=sample` ではAPIキーなしで固定JSONからレポートを生成できる。 |

## 7. データ取得方針

| データ種別 | 取得方法 | MVPで扱う内容 | 扱わない内容 |
| --- | --- | --- | --- |
| 国内ニュース | RSS / News API | タイトル、URL、配信元、公開日時、概要文 | 本文全文、有料記事、ログイン後ページ |
| 海外ニュース | RSS / News API / GDELT等 | タイトル、URL、配信元、公開日時、概要文 | 本文全文、自動翻訳全文保存 |
| マーケット情報 | 株価API | 現在値、前日終値、前日比、変化率 | 自動売買、売買推奨 |
| サンプルデータ | JSONファイル | APIなしでも動作確認できる固定データ | 実運用データとしての利用 |

## 8. レポート出力仕様

生成するMarkdownレポートは、以下の構成とする。

```markdown
# Morning News Report
作成日時: YYYY-MM-DD HH:mm JST

## 1. 今日の注目ポイント

## 2. 国内ニュース

### 経済
- タイトル
  - 配信元:
  - 公開日時:
  - 概要:
  - URL:

## 3. 海外ニュース

## 4. マーケット情報

| 対象 | 現在値 | 前日比 | 変化率 |
| --- | ---: | ---: | ---: |

## 5. 市況コメント

## 6. 注意事項
本レポートは情報提供を目的としており、投資助言ではありません。
```

## 9. 実行モード

本システムは、以下の実行モードを持つ。

| モード | 用途 | データ取得元 |
| --- | --- | --- |
| sample | ポートフォリオ確認用 | `sample_data/*.json` |
| api | 実データ取得用 | API / RSS |

### 9.1 実行モード要件

- `APP_MODE=sample` の場合、APIキーなしでレポート生成できる。
- `APP_MODE=api` の場合、`.env` に設定されたAPIキーを使用してデータを取得する。
- 未設定時は `sample` モードで安全に起動できる。

## 10. 非機能要件

- 実行時間: 朝の定時バッチ1回あたり5分以内を目標とする。
- 可観測性: 成功/失敗件数と最終実行時刻をログで追跡できること。
- 保守性: データソースやカテゴリを設定ファイルで変更可能にする。
- セキュリティ: APIキー等の機密情報は環境変数で管理し、レポートへ出力しない。
- コンプライアンス: 市況コメントは参考情報に限定し、投資助言と誤認される表現を避ける。
- 再現性: APIキーがなくても `sample` モードで主要機能を確認できる。

## 11. 受け入れ条件

- F-01/F-02: 各カテゴリで最低1件以上の記事について、タイトル・URL・配信元・公開日時・概要文を取得できる。
- F-03: 概要文が定義した文字数上限内に整形され、不要改行や重複空白が除去される。
- F-04/F-05: 3指標すべてで現在値と前日比（%）が算出され、欠損時は理由がログに残る。
- F-06: レポートに「タイトル」「URL」「配信元」「公開日時」「短い概要」「マーケット概要」が含まれる。
- F-07: 市況コメントに「本コメントは情報提供であり投資助言ではありません」が必ず表示される。
- F-08: 当日分ファイルが重複せず保存され、再実行時は上書きまたは版管理方針に従う。
- F-09: 失敗時に処理名・発生時刻・原因メッセージを出力する。
- F-10: `APP_MODE=sample` でAPIキーなしにサンプルレポートを生成できる。

### 12.1 公開リポジトリに含めるもの

- ソースコード

- `.env.example`
- `.gitignore`

## 14. 技術スタック

| 区分 | 技術 | 採用理由 |
| --- | --- | --- |
| 言語 | Python 3.x | API取得、ファイル出力、バッチ処理に向いているため |
| HTTP通信 | requests | API/RSS取得をシンプルに実装できるため |
| RSS解析 | feedparser | RSSフィードの記事情報を扱いやすくするため |
| 環境変数管理 | python-dotenv | APIキーを `.env` で安全に管理するため |
| 設定管理 | JSON / Python定数 | カテゴリや対象銘柄を変更しやすくするため |
| レポート出力 | Markdown | GitHub・Obsidian・ポートフォリオで確認しやすいため |
| ログ | logging | 取得失敗や処理状況を記録するため |
| テスト | pytest | 主要処理の動作確認を行うため |
| サンプルデータ | JSON | APIキーなしでもデモ実行できるようにするため |

## 15. 未決事項

- ニュース/価格データの正式ソースと利用規約の確定
- 概要文の文字数上限（例: 80文字 or 120文字）の最終決定
- 市況コメントの定型ルール（閾値・表現テンプレート）の最終決定
- バッチ実行時刻（JST）と再試行回数の最終決定
- レポート再実行時の上書き/履歴保持方針の決定
