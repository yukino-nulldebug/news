01

概要

Morning News は、朝の情報収集にかける時間を減らすため、ニュースとマーケット情報を自動収集し、 日次Markdownレポートとして保存するバッチ型アプリケーションである。

本MVPでは、ニュース本文全文やAIによる高度な要約ではなく、API/RSSから取得できる概要文を整形して掲載する。

MVP方針

API/RSS概要文を整形

掲載情報

タイトル・URL・配信元・公開日時・短い概要

成果物

日次Markdownレポート

02

ポートフォリオで示すこと

  • 要件定義から機能分割、受け入れ条件まで一貫して設計できること
  • API/RSS取得、データ整形、Markdown出力、ログ記録を小さなアプリとしてまとめられること
  • APIキーがない閲覧者でも sample モードで動作確認できること
  • .env やAPIキーを公開しない前提で、GitHub公開に耐える構成にできること
  • 市況コメントは情報提供に限定し、投資助言と誤認されないように扱えること

03

想定ユーザーと課題

想定ユーザー

  • 毎朝、国内外ニュースと市場動向を短時間で把握したい個人ユーザー
  • 業務開始前に「前日から当朝までの要点」を確認したいビジネスユーザー
  • API連携・バッチ処理・Markdown出力の実装例を確認したい閲覧者

解決したい課題

  1. ニュースと株価の関係を同じレポートで確認できない
  2. 複数サイト巡回による情報収集時間が長い
  3. 過去レポートを日付ベースで参照しづらい
  4. APIキーがない環境でも成果物の動作を確認しづらい

04

成功条件

レポート生成

  • 毎朝1回、手動または定時実行でレポートを生成できる
  • 国内ニュース、海外ニュース、マーケット情報が1つのMarkdownにまとまる
  • 各ニュースにタイトル・URL・配信元・公開日時・短い概要が含まれる
  • 日経平均、S&P500、USD/JPY の現在値と前日比を確認できる
  • 市況コメントに「投資助言ではない」注記が表示される
  • 生成されたレポートを過去日付ごとに参照できる

公開・再現性

  • 取得失敗時に処理が完全停止せず、失敗理由がログに残る
  • .env に設定したAPIキーをGitHubへ公開しない
  • APIキーなしでも sample モードでレポート生成を確認できる
  • READMEの手順どおりに第三者がローカルで動作確認できる
  • .env.example に必要な環境変数が明記されている
  • GitHubに機密情報が含まれていない
  • サンプルレポートから、アプリの目的と出力内容が理解できる

05

MVPスコープ

MVPでやる

  • API/RSSで記事情報を取得（タイトル・URL・配信元・公開日時・概要文）
  • 概要文を整形し、短い概要を生成
  • タイトル・URL・配信元・公開日時・短い概要をレポート掲載
  • 株価・指数・為替の取得と前日比計算
  • 市況コメント（参考）を定型ルールで追加
  • 日次Markdownレポート生成と保存
  • sample モードとエラーログを用意

MVPではやらない

  • AIによる高度な要約
  • ニュース本文全文の解析
  • 有料記事やログイン後ページの本文取得
  • ユーザーごとの高度な推薦アルゴリズム
  • 自動売買シグナルの生成
  • 売買推奨や投資助言
  • Web UI・モバイル通知・スマホアプリ化

機能要件

MVP機能一覧
ID 優先度 機能 要件
F-01 MUST 国内ニュース取得 タイトル・URL・配信元・公開日時・概要文を保持する。
F-02 MUST 海外ニュース取得 タイトル・URL・配信元・公開日時・概要文を保持する。
F-03 MUST 概要文整形 不要改行・重複空白を除き、短い概要を生成する。
F-04/F-05 MUST 市況取得・変化率計算 日経平均、S&P500、USD/JPY の現在値・前日比・変化率を扱う。
F-06/F-08 MUST レポート生成・保存 reports/YYYY-MM-DD.md にMarkdownを保存する。
F-07 SHOULD 市況コメント 情報提供に限定し、投資助言と誤認される表現を使わない。
F-09/F-10 SHOULD/MUST ログ・サンプル実行 失敗理由をログに残し、APIキーなしでもサンプル実行できる。

09

実行モード

モード

モード 用途 取得元
sample ポートフォリオ確認用 sample_data/*.json
api 実データ取得用 API / RSS

要件

  • APP_MODE=sample ではAPIキーなしでレポート生成できる
  • APP_MODE=api では .env のAPIキーを使う
  • 未設定時は sample モードで安全に起動できる

10

非機能要件

運用品質

  • 朝の定時バッチ1回あたり5分以内を目標とする
  • 成功/失敗件数と最終実行時刻をログで追跡できる
  • データソースやカテゴリを設定ファイルで変更可能にする

安全性

  • APIキー等の機密情報は環境変数で管理し、レポートへ出力しない
  • 市況コメントは参考情報に限定し、投資助言と誤認される表現を避ける
  • APIキーがなくても sample モードで主要機能を確認できる

11

受け入れ条件

機能別受け入れ条件
対象 条件
F-01/F-02 各カテゴリで最低1件以上、タイトル・URL・配信元・公開日時・概要文を取得できる。
F-03 概要文が文字数上限内に整形され、不要改行や重複空白が除去される。
F-04/F-05 3指標すべてで現在値と前日比（%）が算出され、欠損時は理由がログに残る。
F-06/F-07 レポートに市況コメントと投資助言ではない旨の注意事項が含まれる。
F-08/F-09/F-10 日付付き保存、失敗ログ、APIキーなしのサンプル生成を確認できる。
